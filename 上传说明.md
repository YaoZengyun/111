# 📤 GitHub上传说明

由于网络连接问题，直接推送失败。以下是三种上传方法：

## 🎯 推荐方法：使用GitHub Desktop

**最简单、最稳定的方法！**

### 步骤：
1. **下载GitHub Desktop**
   - 访问：https://desktop.github.com/
   - 下载并安装

2. **登录GitHub账号**
   - 打开GitHub Desktop
   - File → Options → Accounts → Sign in
   - 使用你的GitHub账号登录

3. **添加本地仓库**
   - File → Add local repository
   - 点击 "Choose..." 按钮
   - 选择：`c:\Users\24307\Desktop\anan\QQMessageImageApp`
   - 点击 "Add repository"

4. **推送到GitHub**
   - 点击顶部的 "Publish repository" 按钮
   - Repository name: `111`
   - Description: QQ消息图片助手
   - ☑️ Keep this code private（如果想设为私有）
   - 点击 "Publish repository"

5. **完成！**
   - 等待上传完成
   - 访问 https://github.com/YaoZengyun/111/actions 查看自动构建

## ✅ 上传后的操作

### 1. 查看自动构建
访问：https://github.com/YaoZengyun/111/actions

你会看到：
- ✅ 构建正在运行（黄色圆圈）
- ✅ 构建成功（绿色勾）
- ❌ 构建失败（红色叉）

### 2. 下载APK
构建成功后（约5-10分钟）：

**方法A - 从Actions下载：**
```
Actions → 最新的构建 → Artifacts → app-debug 或 app-release-unsigned
```

**方法B - 从Releases下载：**
```
仓库首页 → Releases → 最新版本 → Assets → 下载APK
```

### 3. 安装APK
1. 下载 `app-debug.apk` 到手机
2. 打开文件管理器
3. 点击APK文件
4. 允许安装未知来源应用
5. 点击安装

## 🔍 故障排除

### Q: GitHub Desktop连接失败？
**A:** 
- 检查网络连接
- 尝试使用VPN
- 检查GitHub服务状态：https://www.githubstatus.com/

### Q: 构建失败怎么办？
**A:** 
1. 点击失败的构建查看日志
2. 常见问题：
   - 网络问题：重新运行构建
   - Gradle权限：已自动修复
   - 依赖下载失败：等待几分钟重试

### Q: 如何更新代码？
**A:** 
修改代码后：
```powershell
git add .
git commit -m "更新说明"
git push
```

或使用GitHub Desktop：
```
1. GitHub Desktop会自动检测更改
2. 填写更新说明
3. 点击 "Commit to main"
4. 点击 "Push origin"
```
